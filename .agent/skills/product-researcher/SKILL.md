# Product Researcher

A skill designed to scan the web for competitor applications, analyze their core features, and suggest actionable UX, UI, and functionality enhancements to make the current product better.

## When to Use This Skill

- User asks to "scan the web for similar apps" or "find competitors".
- User wants recommendations to "enhance the usability" or "make our product better".
- User is planning a new feature and wants to know how industry leaders handle it.

## Prerequisites

- Access to web search capabilities to aggregate related apps.
- Context about the current app's features, tech stack, and target audience.

## Step-by-Step Workflows

1. **Information Gathering**: Use web search to find competitors in the specific niche (e.g., "apps that compare grocery prices and health ingredients").
2. **Feature Extraction**: Identify the core UX patterns used by these alternatives (e.g., scanning flows, list building, health scoring, visual alerts).
3. **Gap Analysis**: Compare the competitor features with our application to identify missing links or areas of friction.
4. **Actionable Suggestions**: Propose 3-5 concrete UX/UI enhancements that address the gap, prioritizing them by positive user impact.
